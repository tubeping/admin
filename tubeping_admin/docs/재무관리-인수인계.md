# 재무관리 인수인계 (최준 대표용)

> 2026-06-04 · 김국태 → 최준. 재무관리 6개 메뉴(`/admin/finance/*`)의 구조·데이터 갱신·남은 과제 정리.

## 1. 현재 구조 한눈에

- **재무관리 6개 메뉴는 모두 신산 Supabase 네이티브** (2026-06-02 전환·배포 완료). 더 이상 Vultr iframe 아님.
  - 화면: `app/finance/*` · 엔진: `app/finance/_components/shinsan-engine.js`
  - API: `app/api/finance/{all,ar-ap,vat}/route.ts` 등 → Supabase `fin_*` 테이블 읽음
- **데이터 테이블(신산 Supabase)**: `fin_sales`(매출)·`fin_purchases`(매입)·`fin_card_tx`(카드)·`fin_bank_in`(입금)·`fin_bank_out`(출금)·`fin_tax_returns`(부가세 신고정본). 스키마: `supabase/migrations/028_finance_tables.sql`
- 접근키는 `.env.local` 에 이미 있음(`NEXT_PUBLIC_SUPABASE_URL`, `SUPABASE_SERVICE_ROLE_KEY`).

## 2. 데이터 최신화 (← 대표님이 직접 하는 부분)

현재 적재 데이터는 **~2026-04-21 스냅샷**까지. 이후분을 반영하는 방법:

```bash
# 1) hub.eumlogics.kr/shinsan 띄운 브라우저 콘솔에서 실행 → 클립보드 복사
copy(localStorage.getItem(Object.keys(localStorage).find(k=>k.startsWith('shinsan_db_'))))

# 2) 복사된 내용을 finance-export.json 으로 저장 (리포 루트)

# 3) 전체 교체로 적재 (중복 안 남 — 갱신 시 반드시 --replace)
node scripts/import-finance-data.mjs finance-export.json --replace
```

- `--replace`: `fin_*` 5개 테이블을 비우고 현재 스냅샷으로 **통째 교체**. 실행 전/후 건수를 출력하므로 확인 가능.
- 플래그를 빼면 기존 위에 **추가(append)** 되어 중복이 생기니, 갱신 때는 항상 `--replace`.
- 적재 후 `tubepingadmin.vercel.app/admin/finance` 에서 수치 확인.

> 참고: 신산이 자체 수집해 둔 홈택스 원본은 `dashboard/scripts/hometax-data_2026-05.json`(2026-05월분) 등에 있음. localStorage export 가 매출·매입·카드·은행을 모두 합친 정본이라 가장 간편.

## 3. 남은 Vultr 의존 — 당분간 그대로 (결정: 2026-06-04)

홈택스 **자동 크롤러**와 **세금계산서 발행**은 아직 이음 Vultr 백엔드(`/api/shinsan/*`)에서 돈다.
- 코드·공동인증서(`SHINSAN_CERT_PATH`)·사업자번호(`SHINSAN_CORP_NUM`)·데이터는 **전부 신산 소유**. 실행 위치만 Vultr.
- 공동인증서 크롤링은 Vercel(serverless) 불가 → **상시 서버 필요**. 당장은 Vultr 유지하기로 결정.
- 추후 신산 명의 서버로 이전 시 옵션·비용·체크리스트: `OneDrive/신산애널리틱스/회사정보/재무관리_Vultr독립_서버옵션비교.md`

## 4. 알아둘 것 (주의)

- 이 앱 API는 basePath 때문에 클라이언트에서 항상 `/admin/api/...` 로 호출(`/api/...` 직접 호출 시 운영 404).
- 미수/미지급은 원본 reconcile 백엔드가 비공개라 재구현분 — 수치 "검증필요" 표기.
- 인증서는 1년마다 갱신 필요(만료 시 자동수집 중단).
